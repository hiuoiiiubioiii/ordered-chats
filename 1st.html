1. Problem Identity

Surface Statement

Given an integer array nums and an integer target, return indices of the two numbers such that they add up to target.

Constraint Meaning

  • Exactly one valid answer exists.
  • You may not use the same element twice.
  • Output is a pair of indices, not a pair of values.

Core Challenge

value relation
→ positional answer
→ preserve original indices while solving efficiently

3. Logical View Mapping

Surface View

Find two numbers
that sum to target.

Structural View

Find indices (i, j)
such that nums[i] + nums[j] = target.

Optimized View

For current x,
need = target - x.
Lookup need in processed prefix.

Main Transformation

pair search
→ complement lookup
→ processed-prefix memory problem

5. Patterns & Tradeoffs

Brute Force Pattern

for each i:
  for each j > i:
    test nums[i] + nums[j]

Time: O(n²)   Space: O(1)

Optimal Pattern

for each i:
  need = target - nums[i]
  if need in map:
    return
  insert nums[i]

Time: O(n) average   Space: O(n)

Tradeoff Law

use extra memory
to eliminate repeated pair scanning

Anti-Pattern

sorting the raw array without preserving original indices
destroys the required output identity

6. Function Diagrams

Scan/Update Function

state_{i+1} = update(state_i, nums[i])

Complement Function

needed_value = target - current_value

Lookup Function

Insert Function

Master Function Flow

(nums, target)

8. Invariant & Proof Layer

Main Invariant

Before processing index i,
seen_map contains all values from indices [0 .. i-1]
mapped to valid earlier indices.

Why a Hit Is Correct

if needed_value is in seen_map,
then an earlier index already satisfies
nums[earlier] + nums[i] = target

Why a Miss Preserves Correctness

if needed_value is absent,
storing current_value extends the processed-prefix summary
for the next iteration

11. Google-Style C++ Prototype

#include <unordered_map>
#include <vector>

class Solution {
 public:
  // Returns the indices of the two distinct values whose sum is equal to
  // target. The problem guarantees exactly one valid answer.
  std::vector<int> twoSum(const std::vector<int>& nums, int target) {
    std::unordered_map<int, int> seen_index_by_value;

    for (int i = 0; i < static_cast<int>(nums.size()); ++i) {
      const int current_value = nums[i];
      const int needed_value = target - current_value;

      const auto it = seen_index_by_value.find(needed_value);
      if (it != seen_index_by_value.end()) {
        return {it->second, i};
      }

      seen_index_by_value[current_value] = i;
    }

    return {};
  }

  // Brute-force baseline for comparison and verification.
  std::vector<int> twoSumBruteForce(const std::vector<int>& nums,
                                    int target) {
    for (int i = 0; i < static_cast<int>(nums.size()); ++i) {
      for (int j = i + 1; j < static_cast<int>(nums.size()); ++j) {
        if (nums[i] + nums[j] == target) {
          return {i, j};
        }
      }
    }

    return {};
  }
};
